## 版本更新记录

### V1.1.0 (2026-04-15) - 重大改进版

**核心优化**
1. **表格分页精度革命性提升**
   - 引入「渲染后测量」方案，将表格真实渲染到隐藏 DOM 中测量每行高度
   - 彻底解决长文本换行导致的分页截断问题
   - 表头高度真实测量（支持表头换行）
   - 合计行高度真实测量，避免间距不一致

2. **组件定位逻辑优化**
   - 支持负 gap（组件重叠），完全保留设计时的相对位置关系
   - 表格跨页后，后续组件精确定位到表格实际底部
   - 新页面第一个组件正确处理与页面顶部的间距

3. **表格渲染改进**
   - 列宽改为均分方案（百分比），避免宽度溢出
   - 单元格文字垂直居中
   - 使用 `min-height` 替代固定高度，允许内容自然撑开

**工程改进**
- iframe 打印使用 `afterprint` 事件 + 5秒兜底，确保可靠清理
- 批量打印使用 `DOMParser` 替代正则提取 body 内容，更健壮
- Decimal.js 合计计算添加友好错误提示
- 表格宽度溢出检测，输出明确错误提示

## 1. 项目概览 (Project Overview)

### 1.1 背景与目标

解决各业务线（电商、医疗、金融）在打印场景中重复开发、维护困难及服务器渲染成本高的问题。

### 1.2 核心策略

* **渲染模式**：利用浏览器算力进行虚拟分页计算，后端不生成 PDF，实现“零服务端压力”。
* **环境依赖**：强依赖 Chromium 内核浏览器 (Chrome / Edge 80+)，确保 CSS Paged Media 渲染高度一致。
* **核心价值**：所见即所得 (WYSIWYG)、插件化扩展、数据与模板深度解耦。

---

## 2. 核心业务流程 (Core Workflow)

1. **接入准备**：业务方提供业务 JSON 数据示例及对应的 **Schema 解释说明**。
2. **模板设计**：
* 在 Web 设计器中导入 Schema，通过可视化资产树拖拽字段绑定组件。
* 配置数据处理管道 (Pipes) 进行格式转换。
* 模板保存为标准 JSON Schema。


3. **SDK 调用**：业务前端加载 `PrintSDK`，传入 `TemplateID` 和业务数据。
4. **计算与预览**：SDK 在隐藏 iframe 中执行虚拟分页算法，处理跨页切割。
5. **执行打印**：调用 `window.print()` 唤起浏览器原生预览，执行打印。

---

## 3. 功能详细说明 (Functional Specifications)

### 3.1 可视化模板设计器 (Template Designer)

* **交互能力**：支持物理标尺 (mm)、网格吸附、智能对齐参考线。
* **布局模式**：
* **流式布局 (Flow)**：组件随内容自动下移，适合文书、清单。
* **绝对定位 (Absolute)**：固定坐标，适合快递面单、票据套打。


* **组件体系 (插件化)**：
* **基础组件**：文本、图片、形状。
* **动态容器**：
* **动态表格**：单层循环，支持表头重复。
* **动态项组**：自动遍历数组，生成 Label-Value 键值对。





### 3.2 数据源对接与绑定 (Data Binding)

* **Schema 字典**：
* **字段定义**：包含路径 (Key)、中文名称 (Label)、类型 (Type)。
* **枚举预设**：支持状态码映射（如 1->待支付），作为推荐配置。


* **数据绑定逻辑**：
* **单点绑定**：处理静态字段。
* **作用域绑定**：在循环容器内，自动切换为行内上下文。


* **数据处理管道 (Pipes)**：支持使用者自主配置 `date`, `currency`, `dict` 等格式化工具。

### 3.3 打印引擎与分页算法 (Print Engine)

* **虚拟分页**：通过 DOM 高度预计算，自动插入分页符。
* **跨页逻辑**：
* **表格拆分**：自动计算单页容量，跨页自动补齐表头。
* **防断裂策略**：单元格/组件高度过载时，整体移至下一页，不进行中间切割。


* **容错处理**：转换失败时输出原始数据 (Raw Data)，不阻断打印流程。

---

## 4. 技术架构 (Technical Architecture)

### 4.1 混合架构设计

* **微内核 + 插件**：排版引擎为核心，具体组件（条码、图表）作为渲染插件载入。
* **隔离环境**：使用 Shadow DOM 确保打印样式与业务方样式不冲突。

### 4.2 技术栈选型

* **核心库**：React / Vue。
* **计算工具**：`lodash-es` (数据), `dayjs` (日期)。
* **图形工具**：`qrcode` (二维码生成, 客户端base64内联), `jsbarcode` (条形码生成, 客户端base64内联)。

---

## 5. 非功能需求 (Non-functional Requirements)

* **性能**：100 页数据从加载到预览耗时 < 3s；SDK 包体积 (Gzip) < 100KB。
* **安全**：数据即用即走，不存储在服务器，符合隐私合规。
* **交互**：设计器需具备自动备份（LocalStorage）和撤销/重做能力。

/**
 * 虚拟分页伪逻辑：计算内容高度并分配页面
 * v1.1.0 改进：支持渲染后测量、负 gap、表格精确分页
 */
async function computePagination(components, maxPageHeight) {
    let pages = [createNewPage()];
    let currentHeight = 0;
    let prevBottom = 0;

    for (const comp of components) {
        // 计算组件间 gap（支持负数，表示组件重叠）
        const gap = comp.yMm - prevBottom;
        
        if (isTable(comp)) {
            // v1.1.0：渲染后测量表格行高
            const { headerHeight, rowHeights, summaryHeight } = 
                await measureTableRowHeights(comp, comp.data);
            
            // 使用真实行高进行分页
            const tablePages = splitTableWithRealHeights(
                comp, rowHeights, maxPageHeight - currentHeight - gap
            );
            
            // 将表格分页结果加入页面
            for (const tablePage of tablePages) {
                if (currentHeight + tablePage.height > maxPageHeight) {
                    pages.push(createNewPage());
                    currentHeight = 0;
                }
                appendToLastPage(pages, tablePage);
                currentHeight += tablePage.height;
            }
        } else {
            const h = await measureHeight(comp);
            const needHeight = isFirstComponent(pages) ? h : gap + h;
            
            if (currentHeight + needHeight > maxPageHeight) {
                pages.push(createNewPage());
                // 新页面第一个组件：应用 gap（保留设计时的相对位置）
                currentHeight = gap + h;
            } else {
                currentHeight += needHeight;
            }
            appendToLastPage(pages, comp);
        }
        
        prevBottom = comp.yMm + comp.height;
    }
    return pages;
}

/**
 * v1.1.0 新增：渲染后测量表格行高
 */
async function measureTableRowHeights(tableComponent, tableData) {
    // 创建隐藏容器
    const container = createHiddenContainer();
    
    // 渲染表格到隐藏容器
    const tableHtml = renderTable(tableComponent, tableData);
    container.innerHTML = tableHtml;
    document.body.appendChild(container);
    
    try {
        // 测量表头高度
        const headerHeight = measureElement(container.querySelector('thead tr'));
        
        // 测量每行高度
        const rowHeights = [];
        container.querySelectorAll('tbody tr').forEach(row => {
            rowHeights.push(measureElement(row));
        });
        
        // 测量合计行高度
        const summaryHeight = measureElement(container.querySelector('tfoot tr'));
        
        return { headerHeight, rowHeights, summaryHeight };
    } finally {
        // 确保清理测量容器
        document.body.removeChild(container);
    }
}
