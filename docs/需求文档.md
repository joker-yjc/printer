## 1. 项目概览 (Project Overview)

### 1.1 背景与目标

解决各业务线（电商、医疗、金融）在打印场景中重复开发、维护困难及服务器渲染成本高的问题。

### 1.2 核心策略

* **渲染模式**：利用浏览器算力进行虚拟分页计算，后端不生成 PDF，实现“零服务端压力”。
* **环境依赖**：强依赖 Chromium 内核浏览器 (Chrome / Edge 80+)，确保 CSS Paged Media 渲染高度一致。
* **核心价值**：所见即所得 (WYSIWYG)、插件化扩展、数据与模板深度解耦。

---

## 2. 核心业务流程 (Core Workflow)

1. **接入准备**：业务方提供业务 JSON 数据示例及对应的 **Schema 解释说明**。
2. **模板设计**：
* 在 Web 设计器中导入 Schema，通过可视化资产树拖拽字段绑定组件。
* 配置数据处理管道 (Pipes) 进行格式转换。
* 模板保存为标准 JSON Schema。


3. **SDK 调用**：业务前端加载 `PrintSDK`，传入 `TemplateID` 和业务数据。
4. **计算与预览**：SDK 在隐藏 iframe 中执行虚拟分页算法，处理跨页切割。
5. **执行打印**：调用 `window.print()` 唤起浏览器原生预览，执行打印。

---

## 3. 功能详细说明 (Functional Specifications)

### 3.1 可视化模板设计器 (Template Designer)

* **交互能力**：支持物理标尺 (mm)、网格吸附、智能对齐参考线。
* **布局模式**：
* **流式布局 (Flow)**：组件随内容自动下移，适合文书、清单。
* **绝对定位 (Absolute)**：固定坐标，适合快递面单、票据套打。


* **组件体系 (插件化)**：
* **基础组件**：文本、图片、形状。
* **动态容器**：
* **动态表格**：单层循环，支持表头重复。
* **动态项组**：自动遍历数组，生成 Label-Value 键值对。





### 3.2 数据源对接与绑定 (Data Binding)

* **Schema 字典**：
* **字段定义**：包含路径 (Key)、中文名称 (Label)、类型 (Type)。
* **枚举预设**：支持状态码映射（如 1->待支付），作为推荐配置。


* **数据绑定逻辑**：
* **单点绑定**：处理静态字段。
* **作用域绑定**：在循环容器内，自动切换为行内上下文。


* **数据处理管道 (Pipes)**：支持使用者自主配置 `date`, `currency`, `dict` 等格式化工具。

### 3.3 打印引擎与分页算法 (Print Engine)

* **虚拟分页**：通过 DOM 高度预计算，自动插入分页符。
* **跨页逻辑**：
* **表格拆分**：自动计算单页容量，跨页自动补齐表头。
* **防断裂策略**：单元格/组件高度过载时，整体移至下一页，不进行中间切割。


* **容错处理**：转换失败时输出原始数据 (Raw Data)，不阻断打印流程。

---

## 4. 技术架构 (Technical Architecture)

### 4.1 混合架构设计

* **微内核 + 插件**：排版引擎为核心，具体组件（条码、图表）作为渲染插件载入。
* **隔离环境**：使用 Shadow DOM 确保打印样式与业务方样式不冲突。

### 4.2 技术栈选型

* **核心库**：React / Vue。
* **计算工具**：`lodash-es` (数据), `dayjs` (日期)。
* **图形工具**：`qrcode` (二维码生成, 客户端base64内联), `jsbarcode` (条形码生成, 客户端base64内联)。

---

## 5. 非功能需求 (Non-functional Requirements)

* **性能**：100 页数据从加载到预览耗时 < 3s；SDK 包体积 (Gzip) < 100KB。
* **安全**：数据即用即走，不存储在服务器，符合隐私合规。
* **交互**：设计器需具备自动备份（LocalStorage）和撤销/重做能力。

/**
 * 虚拟分页伪逻辑：计算内容高度并分配页面
 */
async function computePagination(components, maxPageHeight) {
    let pages = [createNewPage()];
    let currentHeight = 0;

    for (const comp of components) {
        const h = await measureHeight(comp);
        if (currentHeight + h > maxPageHeight) {
            if (isTable(comp)) {
                // 拆分表格行并注入新页，保留表头
                splitAndRedistribute(comp, pages);
            } else {
                pages.push(createNewPage(comp)); // 整体换页
            }
            currentHeight = h; 
        } else {
            appendToLastPage(pages, comp);
            currentHeight += h;
        }
    }
    return pages;
}
